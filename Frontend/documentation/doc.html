Why This Project Was Built

Sahaay was built to simplify interaction using voice and text. It helps learners understand real-world full-stack development by combining frontend, backend, and database.

Project Overview

A full-stack voice assistant using Flask and PostgreSQL with UI + backend integration. Transform your voice into actionable data with secure authentication and persistent storage.

How It Works - User Journey

Step 1: User Registration

  • User visits the application homepage
  • Clicks "Sign Up" to register
  • Enters username, email, and password
  • Backend hashes password and stores in database (public.users table)

Step 2: User Login

  • User enters email and password
  • Backend verifies credentials using password hash comparison
  • Session is created with user_id for authentication
  • User is redirected to profile or chat page

Step 3: Voice Assistant Chat

  • User clicks microphone button on assistant page
  • Browser uses Web Speech API to capture voice input
  • Voice is converted to text automatically
  • Message is sent to backend via /messages/save_message endpoint
  • Backend stores message in public.messages table with timestamp
  • User can also type messages instead of using voice

Step 4: TTS & Voice Logging

  • User's text is processed through /tts/save_text endpoint
  • Voice input is logged in public.tts_logs table
  • Backend converts text to speech and returns audio
  • Browser plays audio response to user

Step 5: View History

  • User navigates to profile page
  • Profile page fetches user's message history from /messages/get_messages/<user_id>
  • User can see all past conversations with timestamps
  • User can also view voice input logs from /tts/get_texts

Database Structure & File Usage

CREATE DATABASE Website_database;

CREATE TABLE public.users (
  id SERIAL PRIMARY KEY,
  username TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  password TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- Used by: auth.py (register/login)
-- Accessed by: form.js (login), profile.js (profile display)

CREATE TABLE public.messages (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES public.users(id),
  message_text TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- Used by: message.py (save/retrieve messages)
-- Accessed by: assist_script.js (chat history), profile.js (message display)

CREATE TABLE public.tts_logs (
  id SERIAL PRIMARY KEY,
  user_text TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- Used by: tts.py (save voice input logs)
-- Accessed by: assist_script.js (voice recording)

public.users: Stores registered user credentials (used by login/register forms)
public.messages: Stores chat messages from users (used by assistant chat)
public.tts_logs: Stores voice input and TTS logs (used by voice assistant)

File Structure & Uses

Backend Files

  • app.py → Main Flask server
    • Initializes Flask application with CORS support
    • Auto-creates database tables on startup (users, tts_logs, messages)
    • Serves the Frontend folder as static files
    • Registers blueprints for auth, tts, and messages
    • Implements security headers (X-Frame-Options, X-Content-Type-Options, etc.)
    • Runs on port 5000 (HTTP server)
  • database.py → Database connection management
    • Uses psycopg2 to connect to PostgreSQL
    • Reads environment variables from .env file (DB_HOST, DB_NAME, DB_USER, DB_PASSWORD)
    • Provides get_conn() function for getting new database connections
    • Enables SSL mode for secure connections to cloud databases
  • auth.py → User authentication module
    • Handles user registration endpoint: POST /auth/register
    • Handles user login endpoint: POST /auth/login
    • Validates email uniqueness and prevents duplicate registrations
    • Uses werkzeug security to hash passwords
    • Returns JWT token or user_id for authenticated sessions
  • message.py → Message handling module
    • Saves user messages: POST /messages/save_message
    • Retrieves user messages: GET /messages/get_messages/<user_id>
    • Stores messages with timestamps in public.messages table
    • Filters messages by user_id for personalized message history
  • tts.py → Text-to-Speech logging module
    • Saves voice/text input: POST /tts/save_text
    • Retrieves all TTS logs: GET /tts/get_texts
    • Stores text input in public.tts_logs table with timestamps
    • Logs all voice conversion requests for history

Frontend-Backend Integration Map

User Authentication Flow

  • register.html + form.js → POST /auth/register (app.py) → stores user in public.users table
  • login.html + form.js → POST /auth/login (app.py) → returns user_id for session
  • profile.html + profile.js → Displays user profile after login

Chat & Messaging Flow

  • assistent.html + assist_script.js → POST /messages/save_message (message.py) → saves to public.messages
  • assistent.html + assist_script.js → GET /messages/get_messages/<user_id> (message.py) → retrieves chat history
  • profile.html + profile.js → GET /messages/get_messages/<user_id> (message.py) → shows message history

Voice & TTS Flow

  • assistent.html + assist_script.js → POST /tts/save_text (tts.py) → saves to public.tts_logs
  • assistent.html + assist_script.js → GET /tts/get_texts (tts.py) → retrieves voice input history
  • Uses Web Speech API for audio capture and playback

Authentication APIs (Register & Login)

All requests must use JSON format in Thunder Client or curl.

Register User

Thunder Client POST request:

POST http://127.0.0.1:5000/register
{
  "username": "newuser",
  "email": "user@example.com",
  "password": "mypassword"
}

curl command:

curl -X POST http://127.0.0.1:5000/register \
-H "Content-Type: application/json" \
-d '{"username": "newuser", "email": "user@example.com", "password": "mypassword"}'

Login User

Thunder Client POST request:

POST http://127.0.0.1:5000/login
{
  "username": "newuser",
  "password": "mypassword"
}

curl command:

curl -X POST http://127.0.0.1:5000/login \
-H "Content-Type: application/json" \
-d '{"username": "newuser", "password": "mypassword"}'
